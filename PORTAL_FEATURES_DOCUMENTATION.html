üîê Security Features

üõ°Ô∏è Authentication & Authorization

1. Password Security

Password Hashing: All passwords are hashed using SHA-256 before storage in database

Password Validation: Strong password requirements enforced:

  • Minimum 8 characters
  • At least one uppercase letter
  • At least one lowercase letter
  • At least one number
  • At least one special character

Password Storage: Passwords never stored in plaintext

2. Session Management

Secure Sessions: Cryptographically secure random session tokens

Session Expiration: Sessions automatically expire after 8 hours of inactivity

Session Validation: All protected routes check session validity before access

Auto-Logout: Users automatically logged out on session expiration

3. Role-Based Access Control (RBAC)

Role Verification: System verifies user role (Student, Lecturer, Admin) before granting access

Portal Separation: Each role has dedicated portals with role-specific features

Access Restrictions: Users can only access features allowed for their role

Admin-Only Features: Critical functions (final semester release) restricted to admin only

4. Registration Code Protection

Lecturer Registration: Requires registration code (LECTURER2026 - change in production)

Code Validation: Case-insensitive code validation

Restricted Access: Prevents unauthorized lecturer account creation

üõ°Ô∏è Input Validation & XSS Prevention

1. Input Sanitization

HTML Escaping: All user inputs are escaped before display to prevent XSS attacks

Script Tag Removal: Removes script tags, event handlers, and malicious content

Safe HTML Rendering: Functions to safely set HTML content without XSS vulnerabilities

Filename Sanitization: Prevents path traversal and malicious filenames

2. Data Validation

Email Validation: RFC-compliant email format checking

Username Validation: Alphanumeric + underscore, 3-20 characters

Input Type Validation: Validates data types before processing

Range Validation: Validates numeric ranges (scores, marks, etc.)

üõ°Ô∏è CSRF Protection

1. CSRF Token Implementation

Token Generation: Cryptographically secure random CSRF tokens

Form Protection: All forms include CSRF tokens

Token Validation: Server validates tokens before processing requests

Token Expiration: Tokens expire after use or time limit

üõ°Ô∏è Rate Limiting & Brute Force Protection

1. Login Rate Limiting

Attempt Limiting: Maximum 5 login attempts per 15-minute window per username

Automatic Reset: Rate limits clear on successful login

Account Protection: Prevents brute force password attacks

üõ°Ô∏è File Upload Security

1. File Validation

File Type Validation: Only allows approved file types (.pdf, .docx, .pptx, images)

File Size Limits: Maximum 10MB per file

Filename Sanitization: Removes dangerous characters and prevents path traversal

MIME Type Checking: Validates actual file type, not just extension

Malicious Content Detection: Scans for potentially dangerous content

üõ°Ô∏è Exam Security Features

1. Exam Integrity

Question Randomization: Questions appear in different order for each student

One-Way Navigation: Students cannot go back to previous questions

Time Restrictions: Auto-submit when time expires

Session Validation: Validates student session during exam

Browser Restrictions: Attempts to prevent cheating during exam

2. Result Protection

Release Control: Results only visible when released by lecturer/admin

Final Grade Control: Final semester grades only visible when released by admin

Access Logging: Tracks when results are accessed

üõ°Ô∏è Database Security

1. SQL Injection Prevention

Parameterized Queries: Supabase uses parameterized queries (prevents SQL injection)

Input Escaping: All user inputs properly escaped before database queries

Query Validation: Validates query structure before execution

2. Data Protection

Encrypted Connections: All database connections use HTTPS

Secure Storage: Sensitive data stored securely in Supabase

Access Control: Database access restricted to authorized users only

üõ°Ô∏è Error Handling & Information Disclosure

1. Secure Error Messages

Generic Errors: Error messages don't reveal sensitive information

User Enumeration Prevention: Login errors don't reveal if username exists

Stack Trace Protection: Production errors don't expose code structure

üõ°Ô∏è Open Redirect Prevention

1. Redirect Validation

URL Validation: Redirect URLs validated to prevent open redirect attacks

Origin Checking: Only allows redirects within same origin

Path Sanitization: Redirect paths sanitized before use
